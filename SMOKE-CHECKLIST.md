# 发版前冒烟测试清单

每次 `npm run ship` 之前手动验证。不通过不发版。

## 1. 导入

- [ ] 拖入巴西展会 Excel → 预览弹窗显示完整链路（原表N个邮箱 → … → N条）
- [ ] 检查弹窗中无邮箱/无公司名/拆分数量正确
- [ ] 含 `/` 或 `//` 的多邮箱单元格自动拆分
- [ ] 预览表所有字段可见（公司名、邮箱、国家、品类等）
- [ ] 未识别列浅色显示在末尾
- [ ] 空邮箱行黄色背景，格式异常行红色背景
- [ ] 点击「保存到联系人」→ 弹窗显示新增/更新/跳过明细

## 2. 联系人

- [ ] 导入后筛选栏出现「⚠️ 异常」标签，数量正确
- [ ] 点击异常标签 → 列出 no_email / invalid_email / no_company 联系人
- [ ] 正常联系人可搜索、可删除

## 3. 收件箱

- [ ] 点击刷新 → 拉取新邮件
- [ ] 单击邮件行 → 选中（灰色左边框）+ 右侧显示详情
- [ ] Ctrl+单击 → 追加选中
- [ ] Shift+单击 → 连续范围选中
- [ ] Ctrl+A → 全选当前筛选视图
- [ ] 多选后右键 → 「删除选中的 N 封」
- [ ] 详情面板发件人信息可选中复制
- [ ] 切换筛选分类正常

## 4. 发送

- [ ] 选择联系人 → 添加到队列
- [ ] 队列正常渲染，可暂停/恢复
- [ ] 发送记录正常显示

## 5. 重启

- [ ] 完全退出后重新打开 → **不弹「阶段追回」提示**
- [ ] 联系人数据不丢失
- [ ] 收件箱缓存不丢失

## 6. 设置

- [ ] 添加账号弹窗正常打开（不报字段缺失）
- [ ] 保存账号 → 测试连接正常

## 7. 命令行

- [ ] `npm test` 全部通过
- [ ] `npm run check` 全部通过
- [ ] `npm run ship` 打包成功
