# Prospecting Email 桌面应用 UI 设计

**日期：** 2026-06-18
**状态：** 已确认
**方案：** Electron 桌面应用（方案 A）

---

## 问题陈述

为 Prospecting Email 工具构建一个独立桌面界面，覆盖客户表导入 → 背调 → 邮件组装 → 发送 → 追踪的完整流程。单人使用，本地运行，工具化定位。

---

## 架构

```
Electron 主进程
├── 脚本桥接层（调用现有 send.js / extract.js / import-csv.js / status.js）
├── IPC 通信层（主进程 ↔ 渲染进程）
└── 系统托盘（最小化后台发送 + 进度通知）

渲染进程（Web UI）
├── 左侧导航（5 页面入口 + 设置）
└── 右侧内容区（根据导航切换）
```

**核心原则：** 主进程执行，渲染进程展示。现有 4 个 Node.js 脚本不改代码，通过 `require` 或 `child_process` 调用。

---

## 页面结构

| 页面 | 功能 |
|------|------|
| 📊 仪表盘 | 今日已发、剩余限额、队列待处理数、最近背调列表、SMTP 连接状态 |
| 📋 客户表 | 拖入 Excel/CSV 解析、飞书 Base 对接、表格展示（公司名/国家/品类/邮箱/背调状态） |
| 🔍 背调详情 | 左侧公司列表 × 右侧信息卡（官网/规模/品类/进口特征/近期动态）→ 底部生成开发信入口 |
| ✉️ 邮件工坊 | 左侧组装区（对象类型/跟进阶段/句库下拉框）、右侧实时预览（双语切换）、加入队列/立即发送 |
| 📤 发送队列 | 队列列表（公司/收件人/主题/状态）、进度条、失败记录、开始/暂停/查看日志 |

---

## 数据流

1. **导入客户表**：拖入文件 → 渲染进程解析（xlsx/CSV）→ 主进程内存存储
2. **公司背调**：勾选公司 → 生成背调请求 → 用户到 Claude Code 完成背调 → 结果回填界面
3. **邮件组装**：选择句库 → 渲染进程实时组装 → 右侧预览双语邮件 → 写入 send-batch.json
4. **发送**：点击开始 → 主进程调用 sendBatch() → 逐封 IPC 回报进度 → 渲染进程更新
5. **追踪**：发送完成 → send-log.json 记录 → 仪表盘/队列页状态刷新

---

## 关键设计决策

1. **模板句库加载方式**：启动时解析 `templates/general-templates.md` → 内存 JSON 对象，下拉框直接绑定
2. **背调方式**：界面生成请求 → 用户到 Claude Code 对话完成背调 → 结果回填（保证背调质量）
3. **发送行为**：关闭窗口 → 最小化到系统托盘后台继续，托盘图标显示进度
4. **文件结构**：Electron 文件放入新 `electron/` 目录，零侵入现有 `send/` `templates/` `docs/`

## 非目标

- 不实现多用户/权限系统（单人使用）
- 不实现自动背调 AI 引擎（背调走 Claude Code 对话）
- 不实现邮件模板在线编辑器（模板维护在 Markdown 文件中）
- 不打包为安装程序（开发模式运行，后续再考虑）
